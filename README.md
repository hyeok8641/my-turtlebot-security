# 🔒 ROS 2 기반 홈 시큐리티 자율순찰 및 AI 화재 경보 로봇 (3조)

본 프로젝트는 터틀봇3 Burger와 스마트폰 Edge AI 기술을 융합한 지능형 방재 순찰 시스템입니다. 실습실 환경을 상시 자율 순찰하며, 화재 위험 요소를 감지하는 즉시 최우선 브레이크 인터럽트 메커니즘을 통해 로봇을 안전하게 정지시키고 중앙 관제 대시보드에 시청각 경보를 동기화합니다.

---

## 👨‍🏫 [교수님 전용] 10초 원터치 라이브 시연 가이드

교수님! 복잡한 터미널 명령어 입력이나 소스 빌드 과정 없이, 발표장에 세팅된 마스터 노트북에서 **아래 딱 3단계 동작만으로 시스템의 모든 핵심 기능을 직접 검증**하실 수 있습니다.

### 1단계: 원터치 시스템 기동 (클릭 1회)
* 바탕화면 터미널 창에 이미 입력되어 있는 아래 명령어를 실행(엔터)해 주십시오. 
* 터틀봇 Core 브링업, Nav2 내비게이션, AMCL 지도 매칭, 3조 마스터 제어 노드가 `tmux` 백그라운드 상에서 재부팅 없이 한 방에 자동 구동됩니다.
```bash
./run_all.sh

2단계: 관제실 대시보드 활성화 (클릭 1회)

    모니터 화면에 자동으로 크롬 브라우저 창(web/index.html)이 팝업됩니다.

    ⚠️ [중요] 크롬 브라우저 보안 규정(Autoplay Restriction) 우회를 위해, 화면의 빈 공간을 마우스로 꼭 1회 클릭해 주십시오. 이 동작을 거쳐야 화재 비상 사이렌 사운드가 정상 출력됩니다.

    활성화 직후, 로봇은 스스로 제자리 2바퀴 자율 회전을 수행하며 초기 위치를 특정(AMCL 정렬)한 뒤, 4개 구역 순찰 주행을 자동으로 개시합니다.

3단계: AI 화재 인터럽트 강제 검증 (마우스 까딱 1회)

    로봇이 순찰 주행을 하고 있을 때, 시연용 스마트폰 카메라 전방에 준비된 화재(Fire) 이미지/라이터 불씨를 제시해 주십시오.

    스마트폰 Edge AI가 이를 포착하는 즉시 0.1초 만에 로봇의 Nav2 주행 Goal을 취소하고 모터를 강제 정지(zero-vector 주입)시킵니다.

    동시에 교수님이 보고 계신 대시보드 화면이 적색으로 반전되며 고주파 비상 사이렌 경보음이 실시간으로 동기화 출력되는 것을 확인하실 수 있습니다.

🛠️ 개발자용 타 환경 최초 구동 및 배포 가이드라인

타 노트북 PC 및 터틀봇3 하드웨어 환경에서 본 패키지를 처음부터 빌드하고 환경을 이식하려는 엔지니어는 아래 설정을 셋팅하십시오.
1. 네트워크 및 타겟 IP 세팅

    각 기기의 터미널에서 hostname -I를 입력하여 IP를 확보합니다.

    run_all.sh 파일을 열어 상단의 환경 변수 선언부를 본인의 실제 IP 환경에 맞춰 갱신합니다.
    Bash

    export ROS_DOMAIN_ID=30  # 3조 지정 도메인 ID
    export ROS_MASTER_URI=http://<노트북_실제_IP>:11311
    export ROS_HOSTNAME=<노트북_실제_IP>

2. 지도 데이터 및 패키지 배포

    노트북 PC의 홈 폴더(~/) 경로 상에 저장소 내부의 my_50_room_map.yaml 및 my_50_room_map.pgm 파일이 위치해 있는지 확인하십시오.

    본 소스 코드를 터틀봇 본체(SBC)의 워크스페이스로 전송한 뒤, 아래 명령어를 수행하여 의존성 빌드를 완료하십시오.
    Bash

    cd ~/turtlebot3_ws
    colcon build --packages-select home_security_pkg
    source install/setup.bash
