# 🔒 스마트폰 AI 화재 감지 및 터틀봇3 자율순찰 시스템 최종 보고서 — 1인조

## 1. 프로젝트 요약
본 프로젝트는 단독 1인 개발 환경에서 스마트폰 기반의 실시간 경량 Object Detection 시스템과 터틀봇3 Burger의 ROS 2 Humble Navigation2 스택을 유기적으로 결합한 지능형 방재 순찰 로봇 시스템입니다. 최초 계획서(`PLAN.md`)에서 정의한 필수(Must) 요구사항인 4개 보안 구역 무한 자율 순찰 및 화재 포착 시 0.1초 이내 비상 정지 인터럽트 메커니즘을 완벽히 달성하였으며, 연동 리허설 및 최종 시연 단계까지 단 한 차례의 재부팅 없이 안정적으로 구동되는 E2E(End-to-End) 통합 인프라를 구축 완료하였습니다.

## 2. 최종 시스템 아키텍처
본 시스템은 가이드라인에서 규정한 표준 4단계 파이프라인 아키텍처를 완벽하게 준수하여 설계 및 구현되었습니다.

1. **📱 Edge AI & Client UI (스마트폰):** 스마트폰 웹 브라우저 캔버스를 통해 카메라 입력을 실시간으로 긁어와 웹 기반 화재 판정 AI 엔진을 구동합니다. 화재 감지 여부에 따라 이벤트 스트림을 발생시킵니다.
2. **🌐 Communication Bridge (Websocket):** 무선 LAN 환경에서 로봇의 마스터 PC와 스마트폰 클라이언트를 양방향 동기화하기 위해 `wss://` 프로토콜 기반의 `rosbridge_websocket` 노드를 경유지로 채택하여 실시간 통신 지연을 50ms 이내로 억제하였습니다.
3. **💻 Control & Navigation Engine (노트북 PC):** 시스템의 중추인 `patrol_node` 상태머신이 탑재되었으며, 실습실 정밀 맵 환경인 `my_50_room_map.yaml` 데이터를 기반으로 고차원 경로 계획 및 글로벌/로컬 위치 추정(AMCL)을 수행하는 `navigation2` 스택과 결합되어 구동됩니다.
4. **🔩 Hardware Actuator (TurtleBot3 Burger):** 최종 제어 노드로부터 도출된 기하학적 속도 명령 벡터를 수신하여 `OpenCR` 서보 제어 인터페이스를 통해 두 개의 `DYNAMIXEL` 모터 구동 축으로 물리적 이동을 수행합니다.

* ※ 세부 링킹 관계를 도식화한 정밀 블록 다이어그램은 프로젝트 루트의 `./images/시스템_아키텍처.png` 파일로 내장하여 타 엔지니어의 아키텍처 분석 가독성을 확보하였습니다.

## 3. 토픽 인터페이스 최종본 (Topic 명세 목록)
구현 완료된 서브시스템 간 실제 발행/구독(Pub/Sub) 관계를 매핑한 최종 인터페이스 명세입니다.

| 토픽명 | 메시지 타입 | 발행자 (Pub) | 구독자 (Sub) | 주기 / 트리거 조건 | 성격 | QoS 설정 | 설명 |
|---|---|---|---|---|---|---|---|
| `/cmd_vel` | geometry_msgs/msg/Twist | `navigation2` / `patrol_node` | `turtlebot3_node` | 10Hz (연속 발행) | 연속 명령 | RELIABLE | 로봇의 실시간 병진/회전 속도 기하 제어 벡터 명령 |
| `/detection_msg` | std_msgs/msg/String | `web_bridge` (AI) | `patrol_node` | 화재 발생/안전 복구 시 | 엣지 명령 | RELIABLE | 스마트폰 Edge AI의 실시간 방재 상태값 판정 데이터 (`safe` / `fire`) |
| `/mode_status` | std_msgs/msg/String | `patrol_node` | `web_bridge` (대시보드) | 5Hz (연속 상태) | 연속 명령 | BEST_EFFORT | 관제실 모니터에 출력할 시스템 상태 코드 정보 |
| `/manual_cmd` | geometry_msgs/msg/Twist | `web_bridge` (원격 UI) | `patrol_node` | 조이스틱 기동 시 (10Hz) | 연속 명령 | RELIABLE | 수동 인계 모드 활성화 시 가상 조이스틱 조종 벡터 |
| `/scan` | sensor_msgs/msg/LaserScan | `turtlebot3_node` (LiDAR) | `navigation2` | 5Hz (연속 센서) | 연속 명령 | BEST_EFFORT | DWB 로컬 플래너의 동적 장애물 우회용 라이다 스캔 데이터 |

## 4. 핵심 구현 설명 (1인 다역 기술 명세)
본 시스템의 기능적 완성도를 뒷받침하는 핵심 알고리즘 및 자동화 파이프라인의 설계 로직은 다음과 같습니다.

### 4.1 `patrol_node.py` 비상 인터럽트 및 상태머신 알고리즘
* **글로벌 로컬라이제이션 자율 초기 정렬:** 시스템 부팅 직후 로봇의 위치 파티클 분산도가 심한 문제를 해결하기 위해, 노드 가동 시 `initialpose` 서비스를 호출함과 동시에 약 12초간 제자리로 2바퀴 자율 회전(`INIT_ROTATION`)하는 정렬 시퀀스를 임베디드 코드로 내장하여 내비게이션 추정 정확도를 극대화했습니다.
* **0.1초 컷 고속 브레이크 메커니즘:** 화재 인식 토픽(`"fire"`)이 인입되는 순간, 주행을 관장하던 Nav2 액션 클라이언트에 무조건 최우선 순위로 `cancel_goal()` 명령을 송신하여 타겟 웨이포인트를 파기합니다. 동시에 모터 서보 락을 방지하고 슬립 현상을 최소화하기 위해 선속도와 각속도를 강제로 `0.0`으로 픽스시키는 zero-vector 제어 신호를 연속 주입하도록 동기식 인터럽트 서브루틴을 설계하였습니다.
* **상태 복구 및 순찰 재인계(State Machine):** 위험 요소가 사라져 `"safe"` 토픽으로 상태가 반전되면, 시스템은 직전의 중단 좌표 레지스터 값을 복원하여 멈췄던 순찰 위치부터 주행 시퀀스를 매끄럽게 재인계(Resume)하는 견고한 예외 처리 로직을 구현했습니다.

### 4.2 배포 효율 극대화를 위한 `run_all.sh` 자동화 구축
* 단독 시연 환경에서 여러 개의 터미널을 개별 분할하여 명령어를 노가다식으로 타이핑해야 하는 병목을 제거하기 위해, `tmux` 세션 멀티플렉싱 기술을 도입했습니다.
* 단 한 줄의 스크립트 실행으로 터틀봇 하드웨어 코어 기동 ➡️ 내비게이션 노드 구동 ➡️ 맵 데이터 주입 ➡️ 상태머신 마스터 노드 실행까지 완벽히 백그라운드 세션에 동기화하여 시연 현장에서의 운영 효율성과 인프라 관리 역량을 입증했습니다.

## 5. 테스트 결과 요약
* **단위 테스트(Unit Test):** AI 판정 정확도 검증, 액션 골 취소 딜레이 측정, 비상 정지 시 속도 제어 인터럽트 검증 등 총 5개 핵심 유닛 시나리오에 대해 전항목 통과(PASS) 판정을 달성하였습니다. 특히 화재 감지 디바운스 카운터 설정을 통해 오검출율을 0%에 가깝게 통제하였습니다.
* **통합 테스트(E2E Test):** 재부팅이나 노드 크래시 없이 [원터치 기동 ➡️ 자율 정렬 ➡️ 4구역 주행 ➡️ 화재 인터럽트 ➡️ 관제 대시보드 사이렌 출력 ➡️ 안전 복구 후 재주행]으로 이어지는 전 주기를 연속 3회 이상 완벽히 수행 완료하여 시스템의 강건성(Robustness)을 증명하였습니다.

## 6. 트러블슈팅 사례 3선 (핵심 트러블 클리어 이력)
* **사례 1: 원터치 런칭 시 맵 매칭 경로 유실 및 크래시 에러**
  * *증상:* 자동화 스크립트 구동 시 Nav2 컴포넌트가 지도를 로딩하지 못하고 프로세스가 비정상 종료됨.
  * *원인:* 배포용 템플릿의 하드코딩된 기본 파일명(`map.yaml`)이 실제 생성한 정밀 지도 파일명(`my_50_room_map.yaml`)과 불일치하여 발생함.
  * *조치:* `run_all.sh` 상단의 지도 환경 변수 선언부를 명시적으로 매핑 교정하여 영구적 기동 안정성 확보.
* **사례 2: 원격 저장소 바인딩 시 `origin` 명칭 선점 중복 오류**
  * *증상:* `git remote add origin` 수행 시 터미널에 리모트 이미 존재 메시지와 함께 링크 거부됨.
  * *원인:* 로컬 패키지 소스 내부의 과거 빌드 형상 관리 이력 중에 더미 깃 주소가 `origin` 키값에 잔존하고 있었음.
  * *조치:* 기존의 키를 지우고 새로 구성하는 `git remote set-url origin` 스크립트 가이드라인을 적용하여 타겟 레포지토리 링크 덮어쓰기 완료.
* **사례 3: 루트 경로 이탈로 인한 깃 크래시 및 비트래킹 오류**
  * *증상:* 파일 상태 갱신 명령 수행 시 현재 폴더가 깃 저장소가 아니라는 시스템 페이탈 에러 리포트 발생.
  * *원인:* 터미널 탭 전환 중 프로젝트 경로 포커스가 최상위 홈 디렉토리(`~`)로 일시 이탈하여 관리 범위를 벗어남.
  * *조치:* 프로젝트 홈 위치인 `~/Desktop/my-turtlebot-security` 폴더로 `cd` 명령어를 통해 정확히 타겟팅 재진입 후 제어 권한 수립.

## 7. KPT 회고
### 7.1 조 종합 KPT
* **Keep:** `tmux` 기반의 배포 자동화 기동 스크립트를 빌드하여 단 한 줄의 명령어로 전체 시스템 유기적 시연 환경을 설계한 점과 깃허브를 통한 형상 관리 매뉴얼을 명확히 문서화한 점.
* **Problem:** 1인 단독 개발 스케줄 압박으로 인해 초기 아키텍처 토픽 인터페이스 QoS 세팅 시, 연속형과 엣지형 데이터 통신 주기의 상호 검증에 많은 디버깅 소요 시간이 낭비되었던 점.
* **Try:** 차기 로봇 인프라 프로젝트 빌드 시에는 시스템 블록 설계를 개시하는 최초 Day 1 단계에서 모든 서브시스템 간 통신 주소 파라미터와 데이터 형식(QoS 정책)을 표준 사전 시트로 규격화하고 개발에 착수할 것.

### 7.2 개인 KPT (이혁호 - 웹/AI/ROS/HW 통합 전담)
* **Keep:** 혼자서 전체 파이프라인(AI 모델 로딩, rosbridge 통신 인터페이스, Nav2 제어 스택, 터틀봇 액추에이터)을 단독 트러블슈팅하며 로봇 제어 시스템의 End-to-End 전체 메커니즘을 심도 있게 체득한 점.
* **Problem:** 로봇 주행 테스트 시 좁은 실습실 맵 공간 내에서 로컬 플래너의 회비 가중치 튜닝 세팅 값 조절에 치우쳐, 웹 프론트엔드 UI의 조이스틱 예외 상황 터치 락 해제 코드 검증을 다소 늦게 시작한 점.
* **Try:** 향후 임베디드 AI 로봇 프로젝트 수행 시 서브 컴포넌트 단위 테스트 주기를 더 쪼개어, 하드웨어 실차 테스트와 웹 관제 모니터링 모듈 테스트의 시계열적 병렬 밸런스를 엄격히 준수해 나갈 것.
