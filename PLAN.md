# 🔒 스마트폰 AI 화재 감지 및 터틀봇3 자율순찰 시스템 계획서 — 3조

## 1. 프로젝트 개요
본 프로젝트는 스마트폰 카메라의 Edge AI(객체 인식) 기술과 터틀봇3 Burger의 ROS 2 Navigation2 시스템을 융합한 3조의 홈 시큐리티 로봇 프로젝트입니다. 로봇은 실습실 내 지정된 4개의 보안 구역을 자율 순찰하며, 스마트폰 카메라를 통해 화재(Fire) 요인이 포착되는 즉시 0.1초 내로 내비게이션 주행을 중단하고 비상 정지합니다. 동시에 웹 기반 중앙 관제 대시보드에 실시간 경보 팝업과 고주파 사이렌을 동기화하여 방재 효율성을 극대화하는 것을 목표로 합니다.

## 2. 기능 요구사항 (MoSCoW 우선순위 정의)
* **Must (필수 구현)**
  * [ROS2/HW] 터틀봇3 Burger 기반 배터리 및 센서 데이터 Bringup
  * [Nav2] 실습실 지도(`my_50_room_map`) 매칭 및 4개 웨이포인트 무한 자율 순찰
  * [AI/ROS2] 스마트폰 독립 카메라 기반 화재(`fire`) 감지 인터럽트 및 모터 0.1초 내 비상 정지
  * [Web] `rosbridge_websocket`을 통한 로봇 상태 수신 및 실시간 시각/청각 비상 경보(사이렌)
* **Should (적정 구현)**
  * [Web] 대시보드 상에서 가상 조이스틱을 이용한 하이브리드 수동 제어 인계
  * [ROS2] 구동 즉시 글로벌 로컬라이제이션 호출 및 제자리 2바퀴 자율 회전(자기 위치 특정)
* **Could (추후 구현)**
  * [AI] 화재 외 외부 침입자(Person) 감지 및 경보 로그 추가
* **Won't (제외)**
  * 로봇 상단에 실제 물리적 소화 액추에이터 탑재 및 분사 구동

## 3. 시스템 아키텍처
* **공통 아키텍처 흐름:** 스마트폰 독립 카메라(AI 추론) ➡️ 웹 프론트엔드(`index.html`) ➡️ wss 통신 ➡️ `rosbridge_server` ➡️ ROS 2 Humble 제어 노드 ➡️ 터틀봇3 Burger 하드웨어 구동

* **주요 ROS 2 노드 설명**
  * `navigation2`: 지도 데이터를 기반으로 글로벌/로컬 경로 계획 및 장애물 회피 주행 담당
  * `patrol_node`: 순찰 상태, 비상 정지 상태, 수동 제어 상태를 제어하는 시스템 핵심 상태머신 노드

## 4. 사용 기술
* **운영체제 및 ROS 버전:** Ubuntu 22.04 LTS / ROS 2 Humble
* **로봇 하드웨어:** TurtleBot3 Burger (Raspberry Pi 4, OpenCR, LDS-01 LiDAR)
* **AI 모델:** 웹 브라우저 기반 실시간 경량 Object Detection (화재 감지) 모델
* **관제 인터페이스:** HTML5 Canvas, 로컬 웹 사운드 합성 엔진, `roslibjs`

## 5. 역할 분담표 (3조 5인 체제 명세)
가이드라인 요구사항에 맞춰 산출물별 책임자를 명확히 분배한 3조의 최종 역할 분담표입니다.

| 서브시스템 | 담당 범위 | 최종 책임자 |
|---|---|---|
| 🌐 웹 프런트 | 대시보드 UI, `roslibjs` 통신 연동, 사운드 락 해제 및 사이렌 구현 | 정현지 |
| 🧠 AI 파이프라인 | 스마트폰 카메라 연동, 화재 감지 디바운스 및 판정 토픽 발행 | 김민상 |
| 🤖 ROS 노드 | `patrol_node` 상태머신 설계, Nav2 액션 클라이언트, 비상 브레이크 제어 | 차재혁 (팀장) |
| 🔩 하드웨어 | 터틀봇3 기동 환경 셋업, LiDAR 데이터 검증, `run_all.sh` 자동화 구축 | 이준제 |
| 🔗 통합·QA | E2E 시스템 인터페이스 연동, 단위/통합 테스트 케이스 수행 및 문서화 총괄 | 이종한 |
